Industrial IT: lets work like its 2022

Industrial IT has developed a lot in my 16 years in the business.

Demand has risen for increased complexity, driven by energy savings, traceability, reduced downtime, and cost savings.
With increased complexity the demand for high quality documentation has also increased.

Specification, standardization and documentation are often not prioritized sufficiently. Lets change that.

To reap the benefits of industrial IT, it must be done right from the beginning.
I have over 16 years of experience with Industrial IT. PLC applications in many different languages and products.
Automatic generation of code and documentation can be utilized to reduce cost and test when building automation solutions.
Either it being a small single plant SCADA system or larger multisite application,
every system can benefit from a programming and visualisation standard
A standard can vary much in complexity but will be a powerfull tool for the developers collaborating on the project.
Troughout a Industrial IT project many learnings will happen in each project phase, Sales phase,
description phase, customer review, programming, unit test, solution test and commissioning.
each learning must be captured, documented and communicated troughout the project team or company.
To help with this task a version control system can be used, but remember that version control systems are only
as good as the effort people using and maintaining them applies.
